# CLS Query — Zhiliao Log Query Plugin

Tencent Cloud CLS (Cloud Log Service) query plugin for the [Zhiliao](https://github.com/git-zhiliao/zhiliao) Agent, providing log search and analysis capabilities.

> [中文版](README.md)

## Features

- **Log Search**: Query logs using CQL syntax with automatic pagination
- **SQL Aggregation**: Support `| select` syntax for statistical analysis
- **Knowledge System**: Three-layer knowledge loading — on-demand doc loading saves tokens
- **Known Topic Mapping**: Reference topics by friendly names instead of UUIDs

## Tools Provided

| Tool | Description | Cost |
|---|---|---|
| `cls-query.search` | Search CLS logs (CQL search / SQL aggregation) | expensive |
| `cls-query.get_topic_knowledge` | Load detailed query pattern docs for a topic on-demand | cheap |

## Directory Structure

```
cls-query/
  config.yaml              # credentials + topic registry (topic_id + region only)
  config.example.yaml      # config template
  src/index.ts             # TypeScript plugin entry point
  package.json             # dependencies (tencentcloud-sdk-nodejs-cls)
  knowledge/               # knowledge directory (gitignored, managed separately)
    CLAUDE.md              # authoring guide for knowledge files
    {topic_name}/
      _catalog.md          # fields, conventions, doc index (always loaded)
      {doc-name}.md        # task-oriented query pattern docs (loaded on-demand)
```

## Knowledge Architecture (3 Layers)

| Layer | Source | Loading | Content |
|---|---|---|---|
| Plugin-level | hardcoded in code | always | CQL/SQL syntax rules, general tips |
| Topic catalog | `knowledge/{topic}/_catalog.md` | always | field definitions, project conventions, doc index |
| Task-based docs | `knowledge/{topic}/{doc}.md` | on-demand | detailed query patterns, analysis recipes, investigation playbooks |

## Agent Guide: Deployment

For agents or operators responsible for deploying the plugin.

### Prerequisites

- Zhiliao Agent runtime (Node.js + tsx)
- Tencent Cloud API credentials (SecretId / SecretKey) from [CAM Console](https://console.cloud.tencent.com/cam/capi)

### Installation

```bash
# 1. Clone into plugins directory
cd agent/plugins/
git clone git@github.com:git-zhiliao/cls-query.git cls-query

# 2. Install dependencies
cd cls-query && npm install && cd ..

# 3. Configure
cp cls-query/config.example.yaml cls-query/config.yaml
# Edit config.yaml with real credentials and topic info
```

### Configuration

Edit `config.yaml`:

```yaml
# Credentials: inject via env vars (recommended) or set directly
secret_id: "${TENCENTCLOUD_SECRET_ID}"
secret_key: "${TENCENTCLOUD_SECRET_KEY}"

default_region: "ap-nanjing"

# Register known topics (only topic_id and region needed)
known_topics:
  my_app:
    topic_id: "your-topic-id-here"
    region: "ap-nanjing"
```

### Verification

```bash
# After starting the Zhiliao Agent, check logs
docker compose logs agent | grep "Plugin loaded"
# Expected: Plugin loaded: cls-query (2 tools)

# If knowledge files are present:
# Knowledge loaded for "my_app": catalog + N docs
```

### Docker Deployment

Mount the plugins directory into the container:

```yaml
services:
  agent:
    volumes:
      - ./agent/plugins:/app/plugins
    environment:
      - TENCENTCLOUD_SECRET_ID=your-id
      - TENCENTCLOUD_SECRET_KEY=your-key
```

## Agent Guide: Knowledge Maintenance

The `knowledge/` directory is gitignored and managed independently from the plugin code. It can be managed as a separate git repo, generated by an external agent at deploy time, or maintained manually.

For the full authoring guide (directory structure, file formats, naming conventions, content layering rules), see [`knowledge/CLAUDE.md`](knowledge/CLAUDE.md).
